# 🎮 PocketQuant

> **Gotta Money 'Em All**

포켓몬 리그 우승에 강한 포켓몬 하나만으로는 부족하다.
타입 상성, 역할 분담, 밸런스를 고려해 **최적의 파티**를 구성해야 한다.

투자도 마찬가지다.

PocketQuant는 **포켓몬 파티 구성을 포트폴리오 최적화 문제로 재해석**하여,
유전 알고리즘(GA, 나아가 NSGA-III)으로 **최적의 전략 조합을 탐색하는 실험 프로젝트**다.

> 📈 **실데이터(yfinance) 백테스트 기반입니다.** 각 전략을 실제 시장 국면(닷컴·금융위기·코로나·금리쇼크)의
> SPY 가격으로 백테스트해 **HP/ATK/DEF/SKILL 4스탯**을 뽑습니다. (투자 자문 아님 — 학습용)

## 🎯 이 프로젝트가 풀고 싶은 문제

최강의 포켓몬은 존재할 수 있다.
하지만 **모든 체육관을 이기는 포켓몬은 존재하지 않는다.**

투자 전략도 마찬가지다.

PocketQuant는 **"가장 강한 전략"** 이 아니라,
**"다양한 시장 국면에서 살아남는 전략 집합"** 을 찾기 위해 만들어졌다.

> 이 문제의식이 단일목적 GA를 넘어 **다목적 최적화(NSGA-III)** 로 향하는 이유다.
> (자세한 정식화는 **[OPTIMIZATION.md](OPTIMIZATION.md)** 참고.)

## 🧬 유전자 (시그널)

전략은 아래 유전자를 조합해 만들어집니다. 각 유전자는 **실제 지표 로직**으로,
가격을 받아 그날의 포지션(0~1, 1=풀매수·0=현금)을 만듭니다. 전략 포지션 = 유전자들의 평균.

| 유전자 | 성격 | 로직 (포지션) |
|--------|------|------|
| `DD`  | 리스크 | 드로다운 스탑 — 고점 대비 −10% 넘게 빠지면 현금화 |
| `RSI` | 심리 | RSI(14) < 70이면 탑승, 과열(군중 심리 과열)이면 현금 |
| `MA`  | 추세 | 가격 > 200일 이평이면 탑승 |
| `BB`  | 변동성 | 볼린저 상단밴드 위(과열)면 현금 |
| `VOL` | 시장 상태 | 실현변동성 레짐 — 평온=탑승 / 중간=반반 / 격동=현금 |
| `MOM` | 추세 강화 | 모멘텀 — 최근 ~3개월 수익률이 양수면 탑승 |

예시 이름: `MA-VOL킹`, `DD-RSI몬`, `DD-MA-MOM몬`, `타이탄 드래곤`

## 🏟️ 체육관 (시장 국면)

각 체육관은 **실제 역사적 기간**을 가리키며, 그 기간의 SPY 가격으로 백테스트합니다.
(`difficulty`/`volatility`는 이제 판정엔 안 쓰는 **연출용 메타데이터** — 실제 난이도는 가격이 직접 만듭니다.)

| 체육관 | 기간 | 근거 |
|--------|------|------|
| `FINANCIAL_CRISIS` | 2007-10 ~ 2009-06 | S&P -57%·시스템 붕괴 → 최대 낙폭 |
| `DOTCOM`           | 2000-01 ~ 2002-12 | -49%(나스닥-78%), 2.5년 느린 약세장 |
| `RATE_SHOCK`       | 2022-01 ~ 2022-12 | -25%, 채권 동반하락이나 질서있음 |
| `COVID`            | 2020-01 ~ 2020-08 | -34%지만 V자 즉시회복(버티면 생존) |

> 💡 코로나는 **변동성 최강이지만 V자 회복**이라 버틴 전략엔 오히려 기회. 닷컴은 **느린 약세장이라 추세추종(MA)이 버티기 좋음.** 이 국면 차이가 스탯에 그대로 찍힙니다.

## ⚔️ 전투 로직 (백테스트 → 스탯)

```
1) 유전자들이 만든 일별 포지션(0~1)을 하루 lag 적용해 시장 수익에 곱함 (룩어헤드 방지)
2) 체육관 기간(gym.start ~ gym.end)만 잘라 자산곡선에서 CAGR / 최대낙폭(MDD) / 샤프 / 평균현금을 측정
3) 0~100 스탯으로 정규화:
     ❤️ HP    = 평균 현금 비중           (위기 버틸 체력)
     ⚔️ ATK   = CAGR                      (-25%→0, +25%→100)
     🛡️ DEF   = 1 − 내MDD/시장MDD         (시장보다 덜 빠진 정도)
     ✨ SKILL = 샤프비율                  (-1→0, 3→100)
```

종족치 `BST = HP+ATK+DEF+SKILL` (0~400). 등급은 적합도(스탯 가중평균, 0~100)로:
`S`(≥90) `A`(≥70) `B`(≥50) `C`(≥30) `D`(그 외)

> 🪙 **"다 꼬라박아도 상폐만 아니면 진 게 아니다"** — 생존/사망 이진판정을 버리고
> 전부 연속 스탯으로 잰다. 페널티(0점 처리)는 없다.

### 데이터 기준

- `data.py`가 yfinance에서 수정종가를 받아 `data_cache/`에 저장합니다.
- `Gym.end`는 **포함 날짜**입니다. yfinance의 `end` 인자는 배타적이라 내부에서 하루를 더해 요청합니다.
- 캐시가 요청 종료일까지 충분하면 오프라인으로 그대로 쓰고, 종료일이 빠진 오래된 캐시는 다시 다운로드합니다.
- `battle.py`는 데이터 I/O 없이 이미 로딩된 가격만 계산하며, 평가 구간 밖 데이터는 스탯에 섞지 않습니다.

## 🚀 실행

> ⚠️ 반드시 **프로젝트 루트**에서 실행하세요. `app/backend/` 내부 파일을 직접 실행하면 상대 import가 깨집니다.
> 한글이 깨지면 (Windows): `$env:PYTHONIOENCODING="utf-8"`
> 의존성: `pip install -r requirements.txt` (yfinance·pandas·numpy). 첫 실행만 SPY 데이터를 받아 `data_cache/`에 캐시 → 이후 오프라인 동작.

### 📖 도감 — 유전자(포켓몬) 설명 보기

```bash
python main.py --dex    # DD몬/RSI몬/.../MOM몬 카드 출력
```

### 단판 모드 — 전략 한 마리 도전

```bash
python main.py          # 유전자 개수 랜덤
python main.py -g 3     # 유전자 3개 고정
```

### 🧬 진화 모드 — 단일목적 GA

```bash
python main.py --evolve
python main.py --evolve --pop 20 --generations 8 --seed 42
```

| 옵션 | 의미 | 기본 |
|------|------|------|
| `--evolve` | 진화 모드 실행 | - |
| `--pop` | 개체군 크기 | 20 |
| `--generations` | 세대 수 | 10 |
| `--seed` | 랜덤 시드 고정 (GA 재현용) | 랜덤 |

#### 출력 예시

```
=== PocketQuant · 진화 모드 (단일목적 GA · 스탯 BST) ===
개체군 16 · 세대 6

[세대  1] 최고적합도  38.8  최강: DD+MA
[세대  2] 최고적합도  42.0  최강: MA
[세대  6] 최고적합도  42.0  최강: MA

=== 최종 챔피언 ===
유전자: MA
이름: MA킹
적합도(스탯 가중평균): 42.0 / 100

스탯블록
    HP   (자본력)   68.5  ##############
    ATK  (공격력)   37.0  #######
    DEF  (방어력)   51.8  ##########
    SKILL(솜씨)     10.7  ##

시장별 강함 (종족치 낮은 순 = 약한 시장):
    RATE_SHOCK          138.6  #######
    DOTCOM              167.7  ########
    COVID               172.4  #########
    FINANCIAL_CRISIS    193.5  ##########
```

> 📌 **이 출력의 핵심:** 가짜 점수 시절엔 "유전자 다 모으기"로 수렴했지만,
> 실데이터에선 진짜 트레이드오프가 생겨 **`MA`(추세추종) 단일로 수렴**합니다 (유전자를 섞으면 엣지가 희석됨).
> 그런데 그 챔피언도 ATK·SKILL은 낮습니다(방어로 번 게 아니라 안 깨진 것). 스탯 가중치를 바꾸면 진화 방향이 바뀌고,
> 이 충돌이 **다목적 최적화(NSGA-III)** 가 필요한 이유입니다.

## 📁 구조

3층 구조: **main(입력) → service(흐름) → backend(계산)**

```
pocket_quant/
├─ main.py              # CLI 입력만 받음 (argparse → service 호출)
├─ requirements.txt     # yfinance · pandas · numpy
└─ app/
   ├─ service.py        # 실행 흐름 조립 (단판/진화 순서 + 시드 + 출력)
   └─ backend/
      ├─ models.py      # dataclass (Stats/Strategy/Gym/Report) + 스탯 가중치 / 등급 테이블
      ├─ data.py        # yfinance 다운로드 + 디스크 캐시 (data_cache/) + LoadedGym 로딩
      ├─ signals.py     # 유전자 = 진짜 지표 로직 → 일별 포지션(0~1)
      ├─ dex.py         # 포켓몬 도감 (유전자 설명 카드)
      ├─ strategy.py    # 전략 생성 + 이름 자동 생성
      ├─ gym.py         # 체육관 4종 (실제 시장 국면 기간)
      ├─ battle.py      # 실데이터 백테스트 → HP/ATK/DEF/SKILL 스탯 (순수 계산)
      └─ evolve.py      # 단일목적 GA (적합도=스탯 가중합)
```
